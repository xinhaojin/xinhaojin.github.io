STP(生成树协议)笔记

一、核心概念与要解决的问题

1. 协议定位

  • 全称：Spanning Tree Protocol
  • 本质：二层链路管理协议，通过逻辑阻塞冗余链路，构建 “无环且有冗余” 的二层网络拓扑。
  • 解决的核心问题：
    1. 广播风暴：消除二层环路导致的数据包无限循环转发。
    2. MAC表震荡：避免因环路引发的MAC地址表频繁刷新，导致设备无法正常转发。
    3. 冗余备份：在主链路发生故障时，备份链路能够自动切换，保障网络高可用性。

2. 核心参数

参数名称 英文全称 定义与构成 作用
桥ID (BID) Bridge Identifier 优先级 (2字节) + MAC地址 (6字节) 选举根桥的核心依据。值越小越优先。
路径开销 Path Cost 端口到根桥的“成本”。速率越高，开销越小。 选举根端口、指定端口的关键依据。
端口ID Port Identifier 优先级 (4 bit) + 端口号 (12 bit) 在开销、BID均相同时，用于打破选举平局。
BPDU Bridge Protocol Data Unit 交换机间交换的协议报文，默认每2秒发送一次。 承载BID、开销等信息，用于STP计算。

注：常用路径开销默认值 (IEEE标准)：10Gbps=2, 1Gbps=4, 100Mbps=19, 10Mbps=100。

好的，我们来对STP（生成树协议）的核心机制做一个清晰、分点的总结。STP的设计堪称网络协议中的经典，其核心思想是 “化繁为简，以控破环”。

STP核心机制总结

1. 终极目标：逻辑上杜绝环路，物理上保留冗余

  • 核心矛盾：二层网络需要物理冗余链路来提高可靠性，但物理环路又会引发广播风暴和MAC表抖动，导致网络瘫痪。
  • 解决方案：STP不是物理上断开线路，而是通过算法逻辑上阻塞（Blocking） 特定端口，将“环状”拓扑转变为“树状”拓扑，同时让被阻塞的端口进入待命状态，在主链路故障时能立即接管。

2. 构建无环树的三大要素

STP通过构建一棵唯一的“生成树”来定义无环路径，这棵树由三个关键角色构成：

  • 根桥 (Root Bridge)

    • 是什么：整个STP域的“根”或“核心”，是生成树的参考点，所有数据的汇聚点。
    • 如何选举：通过比较桥ID（由桥优先级+MAC地址组成），值最小的获胜。
    • 设计原因：为整个网络提供一个唯一的、稳定的中心参考点。所有路径的计算和流量的方向都以根桥为基准，确保了路径的唯一性和无环性。

  • 根端口 (Root Port)

    • 是什么：每台非根桥交换机上有且只有一个。这是该交换机去往根桥路径最优的端口。
    • 如何选举：比较到达根桥的路径开销，最低者胜出。若开销相同，则依次比较发送方的桥ID、端口ID等。
    • 设计原因：为每台非根桥交换机明确指明 “家在哪里”——即如何最快地到达根桥。这保证了所有交换机都知道数据上传的方向，避免了多路径选择带来的混乱和潜在环路。

  • 指定端口 (Designated Port)

    • 是什么：每条链路上有且只有一个。这是被授权向该链路转发数据的“权威出口”。
    • 如何选举：链路两端的设备相互比较，谁有更优的根路径开销，谁的端口就是指定端口。若开销相同，则比较桥ID。
    • 设计原因：为每条链路明确指定 “数据向哪里去” 。它确保了数据只能从离根桥更近的设备（指定端口）流向更远的设备，彻底杜绝了链路上的双向数据传输，这是打破环路的最直接机制。

3. 打破环路的关键：阻塞剩余端口

  • 所有既不是根端口也不是指定端口的端口，都将被置于阻塞（Blocking）状态。
  • 阻塞状态的特点：不转发任何用户数据帧，只接收用于监听的BPDU协议报文。
  • 这就是逻辑上断开环路的实际操作。被阻塞的端口相当于被“禁用”了数据转发功能，从而切断了环路。

4. 算法的基石：BPDU协议报文

  • 是什么：桥协议数据单元，是STP的“选举投票”和“状态沟通”报文。
  • 作用：交换机之间通过交互BPDU来传递根桥ID、根路径开销、自身桥ID等信息，从而完成根桥选举、根端口和指定端口的计算。
  • 设计原因：STP是一个分布式协议，BPDU是所有交换机协同工作的唯一信息来源。“先通过BPDU协商（控制平面），再建立转发路径（数据平面）” 是其核心工作流程。

5. 应对变化：重新计算与收敛

  • 当网络拓扑发生变化（如线缆断开、交换机宕机），检测到变化的交换机会通过BPDU通知整个网络。
  • 所有交换机会重新进行根桥、根端口、指定端口的计算，并调整端口状态（例如，将之前阻塞的端口重新打开为转发状态）。
  • 设计原因：使网络能够从故障中自动恢复，利用冗余路径实现高可用性。这正是部署STP的最终目的。

一句话概括

STP通过选举一个根桥作为核心参考点，让每台交换机找到一个回家的口（根端口），给每条链路指定一个出口（指定端口），然后果断堵上所有其他多余的口（阻塞端口），并依靠BPDU报文来维持这个状态和应对变化，从而在复杂的物理冗余环境中构建出一个逻辑上绝对无环的树形网络。

二、STP (802.1D) 工作流程

STP通过“选举”构建无环拓扑，流程如下：

Step 1: 选举根桥 (Root Bridge) - 网络的参考点

  • 规则：比较所有交换机的BID，值最小的成为根桥。
  • 细节：优先级相同时，比较MAC地址，MAC地址小的获胜。

Step 2: 选举根端口 (Root Port) - 非根桥上去往根桥的最佳路径端口

  • 规则 (优先级从高到低)：
    1. 到根桥的累计路径开销最小。
    2. 对端发送BPDU的交换机BID最小。
    3. 对端发送BPDU的端口ID最小。
    4. 本端端口ID最小。

Step 3: 选举指定端口 (Designated Port) - 链路上负责转发数据的端口

  • 规则 (优先级从高到低)：
    1. 通过本端口所在路径到根桥的开销最小。
    2. 本端口所在交换机的BID最小。
    3. 本端口的端口ID最小。
  • 注意：根桥的所有端口都是指定端口；每条链路上有且只有一个指定端口。

Step 4: 阻塞端口 (Blocked/Alternate Port)

  • 所有既不是根端口也不是指定端口的端口，将被置为阻塞状态，逻辑上断开以防止环路。

STP端口状态机

状态 持续时间 接收BPDU 学习MAC 转发数据 说明
阻塞 (Blocking) - ✓ ✗ ✗ 初始状态，只监听BPDU
监听 (Listening) 15s (Fwd Delay) ✓ ✗ ✗ 参与选举，确定角色
学习 (Learning) 15s (Fwd Delay) ✓ ✓ ✗ 开始学习MAC地址
转发 (Forwarding) - ✓ ✓ ✓ 正常转发数据
禁用 (Disabled) - ✗ ✗ ✗ 端口被管理员关闭

关键点：从阻塞到转发需要30-50秒，这是传统STP收敛慢的主要原因。

三、STP的缺陷与演进：RSTP (802.1w)

STP (802.1D) 的核心缺陷

  1. 收敛缓慢：依赖计时器（Listen + Learn = 30s+），无法快速故障切换。
  2. 被动迟钝：依赖Max Age超时（20s）检测故障，而非主动探测。
  3. 拓扑单一：所有VLAN共享一棵生成树（CST），无法实现负载分担。

RSTP (802.1w) 的改进

RSTP在保持STP基础算法的前提下，实现了秒级收敛（通常<1s）。

1. 端口角色的增强

  • 替代端口 (Alternate Port)：根端口的备份。根端口失效时，立即接管。
  • 备份端口 (Backup Port)：指定端口的备份（多见于共享介质）。

2. 端口状态的简化

将STP的5种状态合并为3种，加速就绪过程。

RSTP状态 等效STP状态 行为
Discarding Disabled, Blocking, Listening 不学习，不转发
Learning Learning 学习，不转发
Forwarding Forwarding 学习，转发

3. 主动快速的收敛机制

  • BPDU作为保活包：所有交换机都每2秒发送BPDU。连续3次收不到邻居的BPDU即认为链路失效，立即触发重新计算，无需等待Max Age超时。
  • P/A机制 (提议/同意)：点对点链路上通过握手瞬间将端口置为转发状态，无需等待计时器。
  • 边缘端口 (PortFast)：连接终端的端口可配置为边缘端口，跳过Discarding和Learning状态，立即转发。（必须搭配BPDU Guard使用:一检测到BPDU报文就禁用端口，防止非法接入交换机）

4. RSTP未解决的问题

  • 所有VLAN仍然运行在同一棵生成树下，无法实现负载分担。

四、终极解决方案：MSTP (802.1s)

MSTP继承了RSTP的快速收敛特性，并通过 “实例” 概念解决了负载分担问题。

MSTP核心思想

1. MST区域 (Region)

  • 由配置了相同三项参数的交换机组成：区域名称 (Name)、修订版本 (Revision)、VLAN-实例映射表。
  • 区域对外呈现为一棵生成树（CST），与外部STP/RSTP交互。

2. MST实例 (Instance)

  • IST (实例0)：默认实例，必须存在，承载所有已映射VLAN，负责与区域外通信。
  • MSTI (实例1-4094)：用户自定义的实例。每个MSTI有独立的根桥、拓扑和端口状态。
  • VLAN映射：管理员将不同的VLAN映射到不同的MST实例中。

MSTP如何实现负载分担

  • 示例：有两条链路Link1和Link2。
    • 将VLAN 10, 20映射到MSTI 1，并设置Switch A为其根桥，引导其流量走Link1。
    • 将VLAN 30, 40映射到MSTI 2，并设置Switch B为其根桥，引导其流量走Link2。
  • 结果：两条链路同时处于转发状态，分别承载不同VLAN的流量，实现了流量的负载均衡。

MSTP优势总结

  • 高效：实例数量取决于网络拓扑复杂度，而非VLAN数量，极大节省资源。
  • 灵活：实现基于VLAN组的精细流量控制和负载分担。
  • 兼容：完美兼容STP和RSTP。

六、总结与演进关系

特性 STP (802.1D) RSTP (802.1w) MSTP (802.1s)
收敛速度 慢 (30-50s) 快 (秒级) 快 (继承RSTP)
端口角色 RP, DP, BP RP, DP, AP, BP 同RSTP
端口状态 5种 3种 同RSTP
核心价值 基础无环 快速收敛 负载分担
VLAN支持 所有VLAN一棵树 (CST) 所有VLAN一棵树 (CST) 多实例，VLAN分组
适用场景 老旧/简单网络 现代通用网络 大型企业/数据中心

技术演进脉络：STP 解决了从无到有（基础无环）-> RSTP 解决了从慢到快（快速收敛）-> MSTP 解决了从用到好用（负载均衡、资源优化）。
