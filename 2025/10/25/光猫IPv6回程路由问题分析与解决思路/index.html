光猫 IPv6 回程路由问题分析与解决思路

一、网络拓扑与背景

在典型的家庭宽带环境中，光猫负责拨号上网并启用 DHCP 功能，为下级路由器分配私有 IPv4 地址。
拓扑结构如下：

光猫开启 IPv6，工作在 Native 模式，路由器通过自动协商（RA、DHCPv6）获取公网 IPv6 前缀并分配给内网设备。
目标需求是：内网服务器能够在公网通过 IPv6 直接访问。

二、问题描述

在部分情况下，路由器能够自动获取 IPv6 前缀，内网设备也能正常通过 IPv6 上网；
但有时路由器获取不到前缀，IPv6 配置不稳定。
更关键的是：当手动为路由器配置静态 IPv6 地址与前缀时，虽然参数与自动模式完全一致，但光猫上却不会出现一条通往路由器内网的 IPv6 静态路由。
结果表现为：

  • 内网设备可以访问外部 IPv6 网站；
  • 但公网无法主动访问内网设备（无回程路由）。

三、原因分析

1. IPv4 与 IPv6 的根本差异

  • IPv4 静态配置可行的原因：
    在 IPv4 网络中，DHCP 服务器与 ARP 协议协同工作，能够自动感知某个地址的存在，网关因此知道去往该地址的路由。

  • IPv6 中的不同点：
    IPv6 不使用 ARP，而采用 邻居发现协议（NDP） 与 路由通告（RA）机制。
    当路由器以 Native（自动）模式 工作时，它会主动向上级（光猫）发送 路由通告（RAdv）或前缀信息（Prefix Delegation, PD），光猫因此自动生成一条指向该前缀的回程路由。

    但当手动配置静态 IPv6 地址时，这种自动上报与通告过程不会发生，光猫就无法得知路由器下方的前缀存在，因此也不会生成回程路由。
    表现上看，路由器的 IPv6 与 LAN 配置都正确，但上级设备“并不知道”这些前缀在哪里。

2. 回程路由的重要性

IPv6 通信的前提是双向可达：

  • 出站路径：从内网 → 光猫 → 公网；
  • 回程路径：从公网 → 光猫 → 内网。

如果光猫上没有回程路由，则公网发往内网的 IPv6 数据包将被丢弃，导致外部无法访问内网设备。
这正是公网 IPv6 无法访问的根本原因。

3. 为什么内网仍能访问外部网站

即使光猫缺少回程路由，内网设备依然可以访问外部 IPv6 网站。
原因在于：

  • 出站方向的路由始终存在；
  • 建立连接时，内网主机主动发起 TCP 握手；
  • 返回的数据包会沿着已有的连接状态表（stateful firewall / NAT66）或临时会话追踪信息返回，无需全局静态回程路由。

因此：

  • 内网访问外部正常；
  • 外部主动访问内网失败。

这一特征也使得问题容易被忽略。

四、问题的本质

IPv6 网络中，自动路由通告（RA/NDP/PD）与上级设备的路由学习机制是保持回程可达性的关键。
当手动配置静态 IPv6 地址时，通告过程被中断，上级网关无法建立回程路由，导致公网到内网的单向通信中断。

五、解决思路（基于 ping 检测 + 重置 Native 状态）

从网络自协商机制出发，保持路由器的 IPv6 自动配置流程是让光猫动态生成回程路由的核心。结合 ping 检测与 Native 模式重置的方案如下：

  1. 以光猫公网 IPv6 可达性为健康指标
    定期（如每 1 分钟）ping 光猫的公网 IPv6 地址（GUA），以此判断 IPv6 通路是否正常：

    • 若可达：说明当前 Native 模式下的路由协商有效，无需干预；
    • 若不可达：判定为路由协商失效（如光猫未生成回程路由、前缀分配异常等），触发修复流程。

  2. 自动重置路由器为 Native 模式
    当检测到光猫公网 IPv6 不可达时，通过脚本自动登录路由器，强制将 IPv6 模式重置为 Native：

    • 触发路由器重新发起 RA（路由通告）和 PD（前缀委托）请求；
    • 促使光猫重新学习路由器内网前缀，并自动生成对应的回程路由；
    • 利用网络协议的自协商能力，重建双向可达性，无需手动配置静态路由。

  3. 历史地址缓存与双重验证

    • 缓存上次成功通信的光猫公网 IPv6 地址，优先使用该地址进行检测，减少对光猫的频繁访问；
    • 若历史地址不可达，再登录光猫获取最新地址并二次检测，避免因临时网络波动导致的误判。

六、结论

  • IPv6 的自动协商机制（RA/PD/NDP）是维持回程路由的核心，Native 模式是触发该机制的最佳方式。
  • 手动配置静态 IPv6 会中断自动协商，导致光猫无法动态学习内网前缀，缺失回程路由。
  • 内网可访问外部 IPv6 网站，但公网无法主动访问内网，本质是“单向路由有效，双向协商失效”。
  • 通过 ping 检测光猫公网 IPv6 可达性，并在异常时重置路由器为 Native 模式，可利用协议自协商能力自动修复回程路由，无需手动配置静态路由。

总结：
IPv6 网络的“自愈能力”依赖于自动协商机制。当网络出现单向可达问题时，无需手动干预路由表，通过重置 Native 模式触发协议重协商，即可让光猫重新学习内网拓扑并生成回程路由。这种“检测 + 重置”的思路，既符合 IPv6 设计的自动化理念，又能在家庭网络环境中稳定解决回程可达性问题。
